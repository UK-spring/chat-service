그렇다면, 왜 AWS S3인가요?

수많은 객체 스토리지 중에서도 S3가 선택받는 이유

💾 과거: 하드웨어의 굴레

예전에는 데이터를 저장하려면 물리적인 서버와 하드 디스크를 직접 구매해야 했습니다.

• 용량 부족: 디스크가 꽉 차면 서버를 끄고 새 디스크를 장착해야 했습니다.
• 비용 낭비: 미래를 대비해 미리 비싼 장비를 사둬야 했습니다(Over-provisioning).
• 데이터 유실: 하드 디스크 고장으로 인한 데이터 손실 위험이 항상 존재했습니다.

DISK FULL
❌ 확장 불가

☁️ 현재: S3의 무한한 확장성

S3는 "인터넷 규모(Internet Scale)" 저장소입니다.

• 무제한 용량: 1바이트부터 수 페타바이트까지, 저장하는 만큼만 자동으로 늘어납니다.
• 내구성: 데이터가 3개 이상의 데이터 센터에 자동 복제되어 99.999999999%(11 9s)의 내구성을 보장합니다.
• 비용 효율: 쓴 만큼만 비용을 지불합니다(Pay-as-you-go).

핵심 개념 (Core Concepts)

Amazon S3를 구성하는 기본 요소들을 시각적으로 이해해봅시다.

🪣 버킷 (Buckets)

버킷은 S3에 저장되는 객체(파일)들을 담는 컨테이너입니다. 윈도우의 폴더와 비슷하지만, 용량 제한이 없고 전 세계에서 유일한 이름을 가져야 합니다.

📄 객체 (Objects)

S3에 저장되는 기본 개체입니다. 데이터(Data), 메타데이터(Metadata), 키(Key)로 구성됩니다. 객체는 불변(Immutable)하므로 수정하려면 전체를 덮어써야 합니다.

S3 요금 구조 (Pricing)

클라우드 비용의 핵심, S3는 어떻게 과금될까요?

💾

1. 저장 용량 (Storage)

저장된 데이터의 크기(GB/TB)와 저장 기간에 따라 과금됩니다. 스토리지 클래스(Standard, IA, Glacier)에 따라 요금이 다릅니다.

📡

2. 요청 및 데이터 검색 (Requests)

PUT, COPY, POST, LIST, GET 등의 API 요청 횟수에 따라 과금됩니다. Glacier 등에서 데이터를 꺼낼 때도 비용이 발생할 수 있습니다.

📤

3. 데이터 전송 (Data Transfer)

S3에서 인터넷으로 나가는 데이터(Outbound)에 대해 과금됩니다. (S3로 들어오는 데이터는 무료입니다)
